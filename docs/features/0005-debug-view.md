# 0005 Debug View

Shipped on 2026-05-25 from PRD `prd/0005-debug-view.md`.

## What it does

Bob gains a third Tauri window (`Bob · Debug`, 1024×700, hidden at boot) toggled with `Cmd+Shift+D` from the Sphere window. The window streams a real-time, color-coded feed of every interesting backend event: user messages, Jarvis thinking start/end, LLM call start/end with prompt and response payloads, sub-task lifecycle, assistant replies, TTS pipeline transitions, and any structlog WARN or ERROR record from a `bob.*` logger. Each event carries a `turn_id` (so all events of a single user turn share a color) and ops with a long body (LLM calls) carry a `correlation_id` pairing their start/end. The frontend supports filter chips for the 7 categories, a severity threshold dropdown, click-to-expand inline JSON payload, click on a turn chip to highlight every event of that turn for 5 seconds, tail-style auto-scroll with a floating `↓ N nouveau(x) event(s)` badge when scrolled up, pause/resume with a pending buffer, and a clear button that wipes the local feed without touching the backend ring buffer.

## Technical surface

- **New backend modules** (`backend/src/bob/`):
  - `debug_log.py` — deep module, no FastAPI coupling. Exposes the `DebugEvent` dataclass (`ts`, `category`, `severity`, `source`, `summary`, `payload`, `turn_id`, `correlation_id`, `replayed`), a `collections.deque(maxlen=2000)` ring buffer, the non-blocking `emit_debug(...)` helper (writes to the buffer and to every subscriber `asyncio.Queue` with a `drop_oldest` overflow strategy), a `subscribe()` async generator that yields the buffer snapshot first (events tagged `replayed=True`) then streams live events, plus a `current_turn_id: ContextVar[str | None]` and a `start_turn() -> str` helper. Installs a `logging.Handler` (`install_structlog_bridge()`) that auto-forwards WARN/ERROR records from `bob.*` loggers to a `system` debug event, while skipping records flagged `_debug_emitted=True` and records originating from `bob.debug_log` / `bob.ws_debug` themselves to avoid double-emission.
  - `ws_debug.py` — thin FastAPI router exposing `/ws/debug`, consuming `subscribe()` and serializing each event to JSON. Cleans up the subscriber queue on disconnect.
- **Backend sprinkle** (no module renamed, just instrumented):
  - `orchestrator.py` — `process_user_message` calls `start_turn()` before its first emit; subsequent emits cover thinking start/end (`decision` / `info`), `_spawn_subtask` (`decision` / `info` with title + goal + task_id), and both reactive + proactive assistant_msg sites (`output` / `info` with speech + ui blocks + proactive flag).
  - `llm_client.py` — both `LMStudioClient.chat`/`.complete` and `ClaudeCliClient.chat` emit paired `llm_call_start` / `llm_call_end` events sharing a locally generated `correlation_id`. Start payload carries the full `messages` array and the model name; end payload carries the response, `latency_ms`, `tokens_in`, `tokens_out`. Exception path emits an `*_end` with `severity="error"` plus the traceback before re-raising.
  - `sub_agent_runner.py` — `_handle_done` (`task`/`info`), `_handle_progress` (`task`/`debug`), `_handle_ask_user` (`task`/`info`), `_fail` (`task`/`warn`).
  - `task_scheduler.py` — `_activate` (`task`/`info` for the actual pending → running promotion site, not `_promote` which doesn't exist) and `cancel` (`task`/`info`).
  - `ws_router.py` — `/ws` connect/disconnect (`system`/`info`), TTS pipeline (`tts_preparing` `voice`/`info`, `tts_ready` `voice`/`debug`, `audio_start`/`audio_end` `voice`/`debug`, `audio_error` `voice`/`warn`).
  - `main.py` — registers the `ws_debug` router next to `ws_router`; installs the structlog bridge at import-time.
- **New frontend modules** (`frontend/src/`):
  - `types/ws-debug.ts` — `DebugEvent` mirror plus `DEBUG_CATEGORIES`, `DEBUG_SEVERITIES`, `SEVERITY_ORDER`, and the `DebugFilters` shape.
  - `hooks/useDebugWs.ts` — owns the `/ws/debug` connection with exponential-backoff reconnect; exposes `{events, paused, setPaused, clear, pendingCount}`. While `paused = true`, incoming frames go into a `useRef<DebugEvent[]>` so a frame arriving on the same microtask as the toggle still lands in the right sink (React batching would otherwise allow a race).
  - `lib/debugFilter.ts` — pure `passesFilters` / `filterEvents` helpers used by `DebugView`.
  - `lib/turnColor.ts` — deterministic `turn_id → HSL hue` via a djb2-style hash, with derived helpers for chip color, highlight background, outline color, and the 6-char short label.
  - `components/debug/DebugView.tsx` — composition root. Owns filter state (local `useState`, not in the hook), the currently highlighted turn id with a single 5-second auto-clear timer, the scroll container ref, and the `isAtBottom` + `newEventsSinceScroll` state driving auto-scroll. Renders the floating `↓ N nouveau(x) event(s)` badge anchored to the bottom and installs the `document.keydown` listener that maps `Space` to pause toggle (skipped when focus is on `<input>` / `<textarea>` / `contenteditable`).
  - `components/debug/DebugToolbar.tsx` — 7 toggleable category chips with a fixed palette (input=blue, llm=purple, decision=cyan, task=green, output=yellow, voice=pink, system=slate), the severity threshold `<select>`, the Pause/Resume button (active visual when paused + inline `+N` pending chip), and the Clear button (label switches to `Clear (N)` when the feed has content).
  - `components/debug/DebugRow.tsx` — memoized per-event row. Owns its `expanded` local state; click on the row body toggles expand and reveals the memoized pretty-printed JSON payload (with an in-house regex tokenizer coloring keys/strings/numbers/bools/null), the `source`, the full `turn_id`, and the `correlation_id` if present. The turn chip renders the first 6 hex chars in its derived color and calls `onTurnClick` with `stopPropagation` so the row does not expand simultaneously.
- **New tests**:
  - Backend: `test_debug_log.py` (28 cases covering envelope, ring buffer cap, snapshot/live ordering, drop-oldest overflow, multi-subscriber, cleanup on close, ContextVar inheritance, structlog bridge filtering), `test_ws_debug.py` (4 WS cases), plus 9 new cases in `test_orchestrator.py` and `test_llm_complete.py` for the orchestrator milestone ordering, turn_id sharing across a turn, sub-task inheritance, and the paired llm start/end + error path.
  - Frontend: `lib/debugFilter.test.ts` (12 cases), `lib/turnColor.test.ts` (9 cases), `hooks/useDebugWs.test.ts` (5 cases for pause buffering, chronological flush on resume, functional setPaused, clear semantics, malformed-frame tolerance).
- **Tauri glue**:
  - `frontend/src-tauri/tauri.conf.json` — declares the `debug` window (`visible: false`, 1024×700, `/?ui=debug`, title `Bob · Debug`).
  - `frontend/src-tauri/src/lib.rs` — `toggle_debug_window` Tauri command toggling `.show()` / `.hide()` on the pre-declared window, registered through `invoke_handler`.
  - `frontend/src-tauri/capabilities/default.json` — widened `windows` from `["main"]` to `["*"]` so the toggle command works from any of the three windows (Sphere, Legacy, Debug). Without this, the existing `legacy` / `new` window labels would also have been rejected.
  - `frontend/src/components/SphereUI.tsx` — `keydown` listener detecting `Cmd+Shift+D` (with the standard `<input>` / `<textarea>` / `contenteditable` guard) that dynamically imports `@tauri-apps/api/core` and calls `invoke('toggle_debug_window')`. Wrapped in `try/catch` so the listener is a silent no-op in the `pnpm dev` web-only context.
  - `frontend/src/App.tsx` — routes `?ui=debug` to `<DebugView />`.

## Notable decisions

- **No-auth, no-redaction by design** — `/ws/debug` is open on `127.0.0.1` and dumps prompts, responses, and exception tracebacks verbatim. Bob is a single-user local app and the debug view is a dev tool, never shipped to an end user. The PRD "Out of Scope" section makes this explicit.
- **Non-blocking emit pipeline** — every subscriber holds its own `asyncio.Queue`. Slow consumers get `drop_oldest` semantics so the orchestrator and LLM call sites never block. A late consumer that reconnects gets the ring buffer snapshot tagged `replayed=True`, so context loss is bounded to the last 2000 events.
- **ContextVar turn propagation** — `current_turn_id` is set via `start_turn()` at the top of `process_user_message`. Sub-tasks spawned via `asyncio.create_task` from inside the turn inherit it natively (Python contextvars standard behavior), which is what reconstructs the full chronology of a turn even after sub-task dispatch. Long-running background tasks unrelated to a user turn (the proactive flusher) intentionally emit with `turn_id=None`.
- **Filter state in the view, not the hook** — `useDebugWs` stays single-purpose (WebSocket lifecycle + buffer + pause). The filter `useState` lives in `DebugView` next to the only consumer (`DebugToolbar`). Putting it in the hook would have widened its surface for no shared consumer.
- **Pause buffer via `useRef`, not `useState`** — the toggle has to be observable to the WebSocket sink on the same microtask the toggle fires, otherwise React's batched updater can let a frame slip into the visible `events` array between `setPaused(true)` and the actual state flip. Using `pausedRef` + `pendingEventsRef` is what the dedicated test in `useDebugWs.test.ts` pins down.
- **In-house JSON tokenizer over `react-json-view`** — a four-token regex (keys / strings / numbers / bools+null) inside a `React.memo`'d `<HighlightedJson>` keeps payload rendering performant for ~30-message LLM dumps without pulling in a dependency. Each row's stringified payload is memoized so toggling other rows doesn't re-stringify.
- **Expand never auto-scrolls** — the scroll snap fires only on `filteredCount` ticking (i.e. a new event arrival), not on row expand or filter changes. This matches the PRD's reading: expanding pushes following rows down, which is fine, but should not yank the user to the bottom.
- **Capability widened from `main` to `*`** — the existing capability targeted a `main` label that no window in this project carries. Switching to a wildcard is what unblocks `invoke()` from `legacy`, `new`, and `debug`. Worth flagging because it relaxes the per-window scoping; tightening it would mean listing the three labels explicitly.
- **`debug_router.py` is unrelated** — the pre-existing `debug_router.py` is an HTTP smoke-test endpoint for Kokoro TTS. The new modules deliberately use distinct names (`debug_log.py`, `ws_debug.py`) so a future cleanup of the smoke-test router doesn't risk touching debug-view code.
